# Basic Security Testing
**Purpose:** Verify security basics before launch — HTTPS, input sanitization, authentication, environment variables, and the minimum security standards.

## The Practice
Launch is where reputations are made or broken. A flawed launch undermines months of work. A clean launch builds trust that lasts.

## The Agency Context
Agency work has constraints that product teams don't — multiple clients, parallel projects, fixed budgets, external deadlines, and the constant tension between quality and speed. Basic Security Testing must account for these constraints to be practically useful.

## When to use
When this is the active constraint in client delivery or agency operations.
