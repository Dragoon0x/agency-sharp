# Mobile Device Testing
**Purpose:** Test on real mobile devices — because simulators miss the touch, performance, and rendering issues that only appear on actual hardware.

## The Practice
Launch is where reputations are made or broken. A flawed launch undermines months of work. A clean launch builds trust that lasts.

## The Agency Context
Agency work has constraints that product teams don't — multiple clients, parallel projects, fixed budgets, external deadlines, and the constant tension between quality and speed. Mobile Device Testing must account for these constraints to be practically useful.

## When to use
When this is the active constraint in client delivery or agency operations.
