# Technical Scoping
**Purpose:** Assess technical complexity during scoping — integrations, custom functionality, performance requirements, and the technical risks that blow up estimates.

## The Practice
Scoping is where money is made or lost. Every hour underestimated is margin that disappears. Every scope gap is a dispute waiting to happen.

## The Agency Context
Agency work has constraints that product teams don't — multiple clients, parallel projects, fixed budgets, external deadlines, and the constant tension between quality and speed. Technical Scoping must account for these constraints to be practically useful.

## When to use
When this is the active constraint in client delivery or agency operations.
