# User Behavior Analysis
**Purpose:** Analyze how users actually use the product — navigation patterns, feature adoption, engagement depth, and the behavioral insights that guide iteration.

## The Practice
Launch is the beginning, not the end. Growth is where launched products become performing products — through data, experimentation, and continuous improvement.

## The Agency Context
Agency work has constraints that product teams don't — multiple clients, parallel projects, fixed budgets, external deadlines, and the constant tension between quality and speed. User Behavior Analysis must account for these constraints to be practically useful.

## When to use
When this is the active constraint in client delivery or agency operations.
