# Agent Team Composition
**Purpose:** Select and configure the right combination of agents for each project — because not every project needs every agent, and the wrong team composition creates overhead without value.

The temptation is to deploy maximum agents on every project. This is usually wrong. More agents means more coordination overhead, more handoffs, more potential for inconsistency. The skill is matching the agent team to the project's actual needs — deploying enough expertise without creating unnecessary complexity.

## The Composition Framework
Tier 1 projects (simple execution — landing page, email template, social assets): 2-3 agents. Designer + Developer. Maybe a content agent. No strategy agent needed if the brief is clear.

Tier 2 projects (standard delivery — website, brand identity, campaign): 4-5 agents. Strategist + Designer + Developer + QA. Content agent if copy-heavy. Each agent has a defined phase and handoff.

Tier 3 projects (complex delivery — product design, full rebrand, multi-channel campaign): 6-8 agents. Strategy + Research + Design + Content + Frontend + Backend + QA + Growth. Requires a coordination agent or human PM to manage the workflow.

## The Coordination Cost
Every additional agent adds coordination cost: another handoff, another context transfer, another potential misalignment. The rule: if an agent's contribution doesn't justify its coordination cost, don't include it. A project doesn't need a growth agent if there's no post-launch growth phase.

## When to use
At project kickoff — deciding which agents to deploy. When projects feel over-staffed (too many agents, too much coordination). When projects feel under-staffed (quality gaps that an additional agent would fill).
