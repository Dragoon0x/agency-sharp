# Agent Library Management
**Purpose:** Maintain and improve the library of agent configurations — versioning, testing, and evolving agents based on project outcomes.

## The Practice
The layer nobody else has built. Individual agents are commodities. The orchestration system that coordinates them into a functioning agency — that's the competitive advantage.

## The Agency Context
Agency work has constraints that product teams don't — multiple clients, parallel projects, fixed budgets, external deadlines, and the constant tension between quality and speed. Agent Library Management must account for these constraints to be practically useful.

## When to use
When this is the active constraint in client delivery or agency operations.
