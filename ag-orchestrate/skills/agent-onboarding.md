# Agent Onboarding to Projects
**Purpose:** Onboard agents to new projects — providing context, constraints, and the project-specific configuration that produces relevant output.

## The Practice
The layer nobody else has built. Individual agents are commodities. The orchestration system that coordinates them into a functioning agency — that's the competitive advantage.

## The Agency Context
Agency work has constraints that product teams don't — multiple clients, parallel projects, fixed budgets, external deadlines, and the constant tension between quality and speed. Agent Onboarding to Projects must account for these constraints to be practically useful.

## When to use
When this is the active constraint in client delivery or agency operations.
