# Multi-Agent Project Management
**Purpose:** Manage projects with multiple agents — tracking progress, resolving blockers, and ensuring coordination across the team.

## The Practice
The layer nobody else has built. Individual agents are commodities. The orchestration system that coordinates them into a functioning agency — that's the competitive advantage.

## The Agency Context
Agency work has constraints that product teams don't — multiple clients, parallel projects, fixed budgets, external deadlines, and the constant tension between quality and speed. Multi-Agent Project Management must account for these constraints to be practically useful.

## When to use
When this is the active constraint in client delivery or agency operations.
