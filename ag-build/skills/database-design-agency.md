# Database Design for Client Projects
**Purpose:** Design databases for client projects — schema design, relationships, indexing, and the data architecture that scales with the client's business.

## The Practice
Engineering is where designs become real. Agency engineering has unique constraints — client codebases, tight timelines, and the need to hand off maintainable work.

## The Agency Context
Agency work has constraints that product teams don't — multiple clients, parallel projects, fixed budgets, external deadlines, and the constant tension between quality and speed. Database Design for Client Projects must account for these constraints to be practically useful.

## When to use
When this is the active constraint in client delivery or agency operations.
