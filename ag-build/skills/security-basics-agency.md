# Security for Agency Projects
**Purpose:** Implement security basics — HTTPS, input validation, authentication, environment variables, and the minimum security standards for professional agency delivery.

## The Practice
Engineering is where designs become real. Agency engineering has unique constraints — client codebases, tight timelines, and the need to hand off maintainable work.

## The Agency Context
Agency work has constraints that product teams don't — multiple clients, parallel projects, fixed budgets, external deadlines, and the constant tension between quality and speed. Security for Agency Projects must account for these constraints to be practically useful.

## When to use
When this is the active constraint in client delivery or agency operations.
