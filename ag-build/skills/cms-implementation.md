# CMS Implementation
**Purpose:** Set up and configure CMS platforms (Sanity, Contentful, Strapi, WordPress) — content modeling, editing experience, and the decisions that determine whether the client can actually manage their own content.

## The Practice
Engineering is where designs become real. Agency engineering has unique constraints — client codebases, tight timelines, and the need to hand off maintainable work.

## The Agency Context
Agency work has constraints that product teams don't — multiple clients, parallel projects, fixed budgets, external deadlines, and the constant tension between quality and speed. CMS Implementation must account for these constraints to be practically useful.

## When to use
When this is the active constraint in client delivery or agency operations.
